    issue-003

    type: "misplacement" severity: "blocking" prompt: "heuristic-to-pattern.v1 (needs review).md" description: > Five rules define pattern lifecycle constraints (scope.type defaults, promotion.candidate behaviour, origin.promoted_from population, sources.signals requirement). These are governance rules about pattern lifecycle and should be centralised. rule_text: > - All generated patterns MUST default to scope.type = "supporting" - Do NOT generate patterns with scope.type = "core" in this step - Always set promotion.candidate = true for heuristic-derived patterns - Always populate origin.promoted_from with the source signal ID(s) - Always include at least one entry in sources.signals related_global_rule: "rule-patterns-born-as-supporting, rule-heuristic-patterns-require-signals" resolution: action: "move" target: "global_rule" explanation: > These rules govern how patterns are created and promoted. They define lifecycle constraints that apply system-wide and should be in knowledge/global/rules/, not embedded in a single prompt. The heuristic-to-pattern prompt should reference the global rules instead.

    issue-004

    type: "misplacement" severity: "warning" prompt: "architecture-intent-generator.v2.md" description: > Multiple governance directives are embedded in the prompt that belong in global rules: applying governance context, and treating rules as constraints not patterns/decisions. rule_text: > - Apply governance rules from the available rule context. - Rules must influence interpretation but must NOT be treated as patterns or decisions. related_global_rule: "rule-precedence-hierarchy, rule-pattern-decision-lifecycle-alignment" resolution: action: "move" target: "global_rule" explanation: > Governance rule interpretation should be centralised.     Notes

    • Governance rules must not be duplicated inside prompts
    • Prompts must defer to global rules for system behaviour
    • Execution rules are allowed and expected within prompts
    • Most prompts (20) have well-separated governance vs execution rules
